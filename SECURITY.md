# Security Policy

## Supported Versions

We currently provide security updates for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| main    | :white_check_mark: |
| < 1.0.0 | :x:                |

## Reporting a Vulnerability

**Do not open a GitHub issue for security vulnerabilities.**

If you discover a potential security vulnerability in this project, please report it by following these steps:

1. Email the maintainer at [jpfreeley@users.noreply.github.com](mailto:jpfreeley@users.noreply.github.com).
2. Provide a detailed description of the vulnerability.
3. Include steps to reproduce the issue.

We will acknowledge your report within 48 hours and provide a timeline for a fix.

## Disclosure Policy

We follow a coordinated disclosure policy. We ask that you do not share the vulnerability publicly until we have had an opportunity to address it.
